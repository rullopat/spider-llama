# Backlog

Work is tracked as one feature story per file.

## Layout

- `done/` — shipped work. Move files here when acceptance criteria are met.
- `todo/` — refined and ready to implement.
- `draft/` — speculative, broad, or under-specified work.

## Story Format

```md
# Title

**Code:** SLLM-000
**Status:** Done | Todo | Draft
**Persona:** Who needs this

## Story

As a [persona], I want [...], so that [...].

## Scope

- ...

## Acceptance

- ...

## Notes

- ...
```

`../PRD.md`, `../ARCHITECTURE.md`, and `../ROADMAP.md` hold the larger product and technical direction. Backlog files are the implementation units. Move files between directories rather than deleting planning history.
